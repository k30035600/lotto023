# 핫 번호 3배 가중치 분석

## 핫 번호에 3배 가중치를 준 이유

### 1. 정성도 점수 공식에서 핫 일치 수의 중요성

정성도 점수 공식:
```javascript
점수 = (핫 일치 수 × 100) - (평균합 거리 × 0.5) - (홀짝 거리 × 10) - (AC 거리 × 5) - (핫/콜 비율 거리 × 15)
```

**핫 일치 수의 가중치가 ×100으로 가장 큼**:
- 평균합 거리: ×0.5
- 홀짝 거리: ×10
- AC 거리: ×5
- 핫/콜 비율 거리: ×15
- **핫 일치 수: ×100** ⭐ 가장 중요

### 2. 상관관계 분석 결과

생성된 번호들의 정성도 점수와 각 요소의 상관계수:
- **핫 일치 수: 0.7456** (매우 강한 양의 상관관계) ⭐ 가장 중요
- 평균합 거리: -0.2511
- 홀짝 거리: -0.1263
- AC 거리: 0.0339

핫 일치 수가 정성도 점수와 가장 강한 상관관계를 가지므로, **핫 번호를 더 많이 선택하면 정성도 점수가 높아진다**는 논리입니다.

### 3. 코드 구현 의도

```javascript
// 통계 기반 가중치 선택: 핫 번호 선택 확률 높이기
const weightedPool = [];
pool.forEach(n => {
    const weight = hotSet.has(n) ? 3 : 1; // 핫 번호는 3배 가중치
    for (let w = 0; w < weight; w++) {
        weightedPool.push(n);
    }
});
```

**의도**: 
- 핫 번호를 3배 더 많이 선택하여 정성도 점수를 높이기
- 핫 일치 수가 높을수록 점수가 높으므로, 핫 번호를 우선 선택

## 문제점

### 1. 실제 통계와의 불일치

**실제 당첨 번호 통계 (최근 100회차)**:
- 핫3:콜3: **34.0%** (가장 많음)
- 핫4:콜2: 25.0%
- 핫2:콜4: 24.0%
- 핫1:콜5: 9.0%
- 핫5:콜1: 7.0%
- **핫6:콜0: 1.0%** (매우 드묾)

**생성된 번호 통계**:
- 핫6:콜0: **98.0%** (거의 대부분)
- 핫5:콜1: 2.0%

### 2. 핫/콜 비율 거리 페널티와의 충돌

정성도 점수 공식에는 **핫/콜 비율 거리 페널티(×15)**가 있습니다:
```javascript
const hotColdDist = Math.abs(hotMatch - refHotAvg) + Math.abs(coldMatch - refColdAvg);
const score = (hotMatch * 100) - ... - (hotColdDist * 15);
```

**문제**:
- 핫 일치 수 가중치(×100)가 핫/콜 비율 거리 페널티(×15)보다 훨씬 큼
- 3배 가중치로 핫6:콜0 조합이 많이 생성됨
- 핫6:콜0은 핫/콜 비율 거리가 6 (|6-3| + |0-3| = 6)
- 하지만 핫 일치 수가 6이면 점수에 +600이 추가되어 페널티(-90)를 상쇄하고도 남음

**계산 예시**:
- 핫6:콜0: 점수 = (6 × 100) - ... - (6 × 15) = 600 - ... - 90 = **510+**
- 핫3:콜3: 점수 = (3 × 100) - ... - (0 × 15) = 300 - ... - 0 = **300+**

핫 일치 수 가중치가 너무 커서 핫/콜 비율 거리 페널티가 효과를 발휘하지 못함.

### 3. 상관계수 계산 불가

핫/콜 비율 거리가 모두 0.00으로 동일하여 상관계수를 계산할 수 없습니다:
- 생성된 번호들이 대부분 핫6:콜0 조합
- 핫/콜 비율 거리 = |6-3| + |0-3| = 6 (모두 동일)
- 변동이 없어 상관계수 계산 불가

## 개선 방안

### 방안 1: 가중치 조정

**현재**: 핫 번호 3배 가중치
**제안**: 핫 번호 1.5~2배 가중치

**이유**:
- 핫 일치 수 가중치(×100)가 이미 충분히 큼
- 추가 가중치를 줄여서 핫/콜 비율이 더 균형잡히도록

**예상 효과**:
- 핫3:콜3, 핫4:콜2, 핫2:콜4 조합 비율 증가
- 핫6:콜0 조합 비율 감소

### 방안 2: 핫/콜 비율 거리 페널티 강화

**현재**: 핫/콜 비율 거리 ×15
**제안**: 핫/콜 비율 거리 ×30~50

**이유**:
- 핫 일치 수 가중치(×100)와 균형을 맞춤
- 핫6:콜0 조합에 더 큰 페널티 부여

**계산 예시**:
- 핫6:콜0 (페이널티 ×30): 점수 = 600 - ... - 180 = **420+**
- 핫3:콜3 (페이널티 ×30): 점수 = 300 - ... - 0 = **300+**
- 여전히 핫6:콜0이 유리하지만 차이가 줄어듦

### 방안 3: 핫/콜 비율 제약 추가

**제안**: 생성 단계에서 핫/콜 비율 제약 추가

```javascript
// 핫/콜 비율 제약: 핫3:콜3, 핫4:콜2, 핫2:콜4만 허용
const allowedRatios = [
    { hot: 3, cold: 3 },
    { hot: 4, cold: 2 },
    { hot: 2, cold: 4 }
];

// 생성된 조합이 허용된 비율인지 확인
const hotCount = selected.filter(n => hotSet.has(n)).length;
const coldCount = selected.filter(n => coldSet.has(n)).length;
const isValidRatio = allowedRatios.some(r => r.hot === hotCount && r.cold === coldCount);
if (!isValidRatio) continue;
```

**이유**:
- 실제 통계에 맞는 비율만 생성
- 핫6:콜0, 핫1:콜5 등 극단적 조합 배제

### 방안 4: 가중치 제거 + 핫/콜 비율 거리 페널티 강화 (권장)

**제안**:
1. 핫 번호 가중치 제거 (1배로 변경)
2. 핫/콜 비율 거리 페널티 강화 (×30~50)

**이유**:
- 핫 일치 수 가중치(×100)만으로도 충분히 핫 번호를 선호함
- 추가 가중치 없이도 핫 번호가 자연스럽게 많이 선택됨
- 핫/콜 비율 거리 페널티를 강화하여 균형잡힌 비율 유도

**예상 효과**:
- 핫3:콜3, 핫4:콜2, 핫2:콜4 조합 비율 증가
- 핫6:콜0 조합 비율 감소
- 실제 통계와 일치하는 생성 패턴
