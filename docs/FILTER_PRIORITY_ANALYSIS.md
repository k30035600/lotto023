## 각 필터의 특성 분석

### 1. AC 필터

**특성:**
- 제약 강도: **높음** (조합의 산술적 특성)
- 목표 값: AC 8 (35% 확률)
- 제약 효과: 극한 조합(AC 5 이하)을 먼저 배제
- 실제 분포: AC 8이 35%로 가장 많음

**장점:**
- AC 5 이하는 5.85%로 매우 드묾 → 먼저 배제하면 유효한 조합만 남음
- 조합의 산술적 복잡도를 제한하여 후속 필터 적용 시 효율적
- 정성도 점수에서 AC 거리 페널티가 있음 (×5)

**단점:**
- AC값은 조합 특성이므로 개별 번호 선택 시 직접 제어 어려움
- 하지만 AC 5 이하 배제는 명확한 제약

### 2. 홀짝 필터

**특성:**
- 제약 강도: **중간** (홀수 23개, 짝수 22개)
- 목표 비율: 홀3:짝3 (33.4% 확률)
- 제약 효과: 번호 풀을 약 50%로 축소
- 실제 분포: 홀3:짝3이 가장 많음

**장점:**
- 번호 풀을 명확하게 2분할하여 후속 필터 적용 시 효율적
- 정성도 점수에서 홀짝 거리 페널티가 있음 (×10)
- 개별 번호 선택 시 직접 제어 가능

**단점:**
- 핫/콜 필터와 비교 시 제약 강도가 비슷함

### 3. 핫/콜 필터

**특성:**
- 제약 강도: **중간** (22개 핫, 23개 콜)
- 목표 비율: 핫3:콜3 (34% 확률)
- 제약 효과: 번호 풀을 약 50%로 축소
- 실제 분포: 핫3:콜3이 가장 많음

**장점:**
- **정성도 점수에서 핫 일치 수가 가장 큰 영향** (×100, 상관계수 0.7456)
- 핫/콜 비율 거리 페널티가 있음 (×15)
- 번호 풀을 명확하게 2분할

**단점:**
- 핫/콜 분류는 회차별로 달라질 수 있음 (동적 특성)
- 하지만 저장 회차 기준으로 고정하면 일관성 유지 가능

## 실무적 고려사항

### 1. 정성도 점수와의 일치도

핫 일치 수가 정성도 점수와 가장 강한 상관관계(0.7456)를 가지므로, **핫/콜 필터를 우선 적용하는 것도 합리적**입니다.

하지만 실험 결과를 보면:
- **AC → 홀짝 → 핫/콜**: 9.26% 성공률
- **핫/콜 → AC → 홀짝**: 9.15% 성공률

차이는 미미하지만(0.11%p), AC 필터를 먼저 적용하면 극한 조합을 먼저 배제하여 더 안정적입니다.

### 2. 필터 제약 강도

- **AC 필터**: 조합 특성 (높은 제약)
- **홀짝 필터**: 번호 풀 2분할 (중간 제약)
- **핫/콜 필터**: 번호 풀 2분할 (중간 제약)

AC 필터는 조합 특성이므로 먼저 적용하면 유효한 조합만 남겨서 후속 필터 적용 시 효율적입니다.

### 3. 정성도 점수 가중치

정성도 점수 공식:
```
점수 = (핫 일치 수 × 100) - (평균합 거리 × 0.5) - (홀짝 거리 × 10) - (AC 거리 × 5) - (핫/콜 비율 거리 × 15)
```

가중치 순서:
1. 핫 일치 수: ×100 (가장 중요)
2. 핫/콜 비율 거리: ×15
3. 홀짝 거리: ×10
4. AC 거리: ×5
5. 평균합 거리: ×0.5

하지만 가중치만으로 우선순위를 결정하기보다는, **필터 제약 강도와 실험 결과를 종합**하여 결정하는 것이 합리적입니다.
