# Perfect 정렬·품질(기대) 관련 정리

## 채택한 품질 방법 (요약)

1. **지표 분리**: Perfect = 통계 적합도(정렬·취향). **당첨 적중 참고**는 별도(백테스트·리포트).
2. **롤링**: 회차별 평가 시 **해당 회차 당첨만** 쓰고, 알고리즘·비교용 과거 창은 문서/스크립트로 고정(누수 주의).
3. **무작위 대조**: `utils/quality_report_rolling.py`로 **무작위 기준선** 생성 (`docs/QUALITY_REPORT_ROLLING.md` 참고).
4. **출시**: 수치 **강제 게이트**보다 **리포트·고지** 우선.

## 현재 Perfect 정렬이 보장하지 않는 것

- **3등(당첨번호 5개 일치)** 과의 직접 연관은 없습니다.
- 미추첨 시 점수는 **「선택한 통계 구간의 핫 22개와 겹치는 개수」**와 **「그 구간 6개 번호 합의 평균에 가까운 정도」**입니다.
- 로또는 무작위 추첨이므로, 과거 통계에 맞춘 조합이 **미래 당첨과 많이 겹친다**는 뜻이 아닙니다.

## “시장 개방” 수준의 품질을 맞추려면 (개선 아이디어)

1. **지표 분리**  
   - *통계 적합도*(현 Perfect)와 *역사적 백테스트*(과거 각 회차에서 이 조합이 몇 개까지 맞췄는지 분포)를 **별도 표시**해 사용자 기대를 맞춥니다.

2. **행운 vs 조회기간**  
   - 행운은 화면 **조회기간**(`currentStatsRounds`)을 씁니다. 저장 파일에 그 구간이 남지 않으면, 나중에 **오프라인 스크립트**와 완전 동일한 순위를 재현하기 어렵습니다.  
   - 개선: 저장 시 조회 회차 범위(또는 yy/mm/dd)를 **메타 컬럼**으로 기록.

3. **상위 N장 다양성**  
   - Perfect 상위 10이 서로 비슷한 번호만 남으면 체감 품질이 떨어질 수 있습니다.  
   - 개선: 상위권에서 **번호 집합 유사도(자카드 등)** 페널티를 두어 분산된 조합을 남기는 옵션.

4. **자동 “출시” 조건**  
   - “상위 10에 3등급 패턴 1개 이상” 같은 규칙은 **정의를 수식으로 고정**(예: 백테스트 N회 중 최대 4개 이상 일치 비율)하고, **무작위 대조군**과 비교하지 않으면 의미가 애매합니다.
